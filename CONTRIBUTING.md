# Contributing

Issues and discussion are welcome.

For code contributions, note the project's standing rule: **changes intended
to affect playing strength are only merged with statistical evidence** — an
SPRT result (or equivalent games-based test) against the current release at a
stated time control. Speed-only changes must demonstrate node-identical
behavior (fixed-depth node counts unchanged) plus a measured wall-clock win.
`docs/QUALITY_GATES.md` describes the validation methodology.

Bug reports with a reproducing FEN/position are gold.
